# MVP 0.1.3：Stable Step Model Refactor

## 背景

当前业务流程录制把 Playwright recorder 的低层 action index、UI step id、展示顺序、用户编辑态、删除/插入状态和循环片段引用混在 `mergeActionsIntoFlow` 中反复重建，导致继续录制、停止录制、删除步骤、插入步骤和编辑循环片段时步骤容易错乱。

## 目标

- `FlowStep.id` 创建后保持稳定，不再随数组位置重排。
- `FlowStep.order` 只表示当前展示顺序，可以随时重算。
- recorder action index 不再作为长期身份。
- 新录制 action 只生成新 step，不覆盖旧 step 的用户编辑态。
- 删除/插入/继续录制/从中间录制不破坏原 step id。
- typing/fill 噪声压缩为稳定业务步骤。
- repeat segment 只引用 stable step ids，显示范围按当前 order 计算。
- 导出 JSON/YAML 时剥离内部 recorder state 和 legacy artifacts。

## 非目标

- 不改 Playwright recorder/player 核心。
- 不引入 Redux/XState 等重型状态框架。
- 不实现 Native Messaging、Runner、AI 生成、CI。

## 方案

采用 Web GPT Pro 建议的 B 方案：

```text
保留 steps: FlowStep[] 作为 UI projection
新增 artifacts.recorder.actionLog
FlowStep.id 改为稳定身份
FlowStep.order 只做展示顺序
新增 sourceActionIds 连接 step 与 action log
逐步替换 mergeActionsIntoFlow 内部实现
```

## 实施清单

- [x] 扩展类型：`FlowRecorderState`、`RecordedActionEntry`、`RecordingSession`、`FlowStep.kind/sourceActionIds`。
- [x] 新增稳定 ID 与迁移工具：补齐 recorder state、迁移旧 flow、重算 order。
- [x] 替换 merge：从 action payload 提取新增 action suffix，append 到 action log，再投影为新 steps。
- [x] 重写删除/插入：只操作 stable steps，不再写 deletedActionIndexes 或负数 action index。
- [x] 稳定 repeat segment：清理悬挂 step/parameter，范围按当前 order 展示。
- [x] 清理导出：剥离 `artifacts.recorder` 与 legacy bookkeeping。
- [x] 纯函数测试：`npm run test:flow --prefix ./examples/recorder-crx`。
- [x] 构建验证：`npm run build --prefix ./examples/recorder-crx` 和必要的根构建验证。

## 验收

- 删除中间步骤后，后续步骤 id 不变。
- 插入空步骤后，原步骤 id 不变，新步骤使用新的 stable id。
- 继续录制不覆盖旧步骤，停止后新增步骤不消失。
- 从某一步后继续录制，只插入新步骤，不丢失后续旧步骤。
- 编辑 repeat rows 不改变 repeat stepIds，显示不出现 `s012-s011`。
- 输入框连续 fill/typing 合并为一个业务 step。
- 导出的 JSON/YAML 不包含内部 recorder state。

## 测试用例

新增 `examples/recorder-crx/src/flow/stepStability.test.ts`，覆盖：

- 继续录制只追加新 action，并保留用户编辑的 step intent。
- 删除步骤后，下一次完整 recorder payload 不会让旧步骤复活。
- 插入空步骤不重编号已有 step，且 manual step 不绑定 action。
- 从中间插入录制时，后续原步骤不丢失，下一批 action 插在上一批之后。
- 连续 fill/typing press 压缩为一个业务 step。
- recorder 原地更新 fill action 时，同步刷新已有 step 的最终值和 sourceCode。
- 用户手工编辑过的 value assertion 不会被 recorder 原地刷新覆盖。
- CapsLock、Shift、Backspace、Space、Arrow、Tab、单字符输入等输入噪声会合并到同 target fill。
- 不同 target 的 press/fill 不会误合并。
- 删除后的 fill step 即使对应 recorder action 原地刷新也不会复活。
- 同一输入框跨 payload 继续输入时只保留最终输入值，后续不同字段保持独立业务 step。
- 删除 repeat segment 内步骤时，悬挂 stepIds/parameters 被清理。
- 导出清理会剥离 recorder action log 和 legacy bookkeeping，compact YAML 不泄漏内部 artifacts。
